# M4 Evidence Contract

Status: EP-001 baseline
Policy version: 1

## Layout

Public M4 proof uses repo-relative paths only.

| Zone | Purpose |
|---|---|
| `docs/m4-public-proof-report.md` | Human-readable M4 report. EP-001 creates the scaffold, later stories fill the proof sections. |
| `docs/m4-evidence-contract.md` | This contract: layout, manifest fields, redaction policy and validation commands. |
| `docs/m4-compatibility-matrix.md` | Human-readable compatibility matrix rendered from the machine-readable EP-002 data. |
| `docs/m4-benchmarks-and-memory.md` | Human-readable benchmark, memory and threshold report generated for EP-003. |
| `docs/m4-replay-and-dogfood-demo.md` | Human-readable EP-004 replay corpus, public event stream and Paneflow dogfood demo procedure. |
| `evidence/m4/evidence-manifest.json` | Machine-readable registry for public M4 artifacts. |
| `evidence/m4/compatibility-matrix.json` | Machine-readable terminal behavior matrix with fixture links, source references and platform measurements. |
| `evidence/m4/m4-benchmark-summary.json` | Deterministic M4 measurements for ingest, snapshot, replay and comparison. |
| `evidence/m4/m4-memory-profile.json` | 10k, 100k and 1M logical-line memory profiles with Hera-owned counters. |
| `evidence/m4/m4-performance-thresholds.json` | Maintainer-authored threshold policy for M4 performance evidence. |
| `evidence/m4/m4-performance-report.json` | Machine-readable rollup created by evaluating benchmark and memory evidence against thresholds. |
| `evidence/m4/m4-replay-verification.json` | Machine-readable EP-004 deterministic replay verification for the public corpus. |
| `evidence/m4/replay-event-streams/*.jsonl` | Public Hera-specific JSONL event streams exported from scrubbed or synthetic replay fixtures. |
| `evidence/m4/paneflow-dogfood-demo-*.json` | Public Paneflow feature-gate and shadow demo summaries, never raw local captures. |
| `evidence/m4/*.json` | Small public evidence summaries, never raw terminal captures. |
| `crates/terminal-fixtures/fixtures/m4-replay/*.json` | Scrubbed or synthetic public replay fixtures. |
| `crates/terminal-fixtures/fixtures/` | Scrubbed or synthetic replay and fixture inputs that can be validated by Rust tests. |
| `crates/*/benches/` | Criterion harnesses owned by their crate, not bulky Criterion output. |

Local raw dogfood artifacts stay out of this repo's public proof package. Use a
local directory such as `<paneflow-repo>/.paneflow-audit/hera-dogfood/` and copy
only scrubbed summaries or deterministic generated fixtures into Hera.

## Manifest Fields

Every public artifact listed in `evidence/m4/evidence-manifest.json` must have:

| Field | Rule |
|---|---|
| `id` | Stable unique identifier. |
| `type` | Human-readable artifact kind such as `report`, `redaction_policy` or `dogfood_smoke_summary`. |
| `path` | Slash-separated repo-relative path. Absolute paths and parent segments are invalid. |
| `source_command` | Command or manual source used to generate the artifact. |
| `generated_at` | UTC timestamp ending in `Z`. |
| `redaction_checked_at` | UTC timestamp ending in `Z`, not older than the current redaction policy. |
| `privacy` | `public_summary`, `scrubbed_public`, `local_private` or `raw_local`. `raw_local` is rejected in the public manifest. |
| `reproducible` | `true` when the artifact can be regenerated by command. |
| `non_reproducible_reason` | Required when `reproducible` is `false`. |

If an artifact cannot be regenerated, it must be called out as
non-reproducible with a reason. It cannot be presented as proof without that
qualification.

## Redaction Policy

Public artifacts must not copy:

- raw prompts or private terminal transcripts
- local home paths or usernames
- known API key markers or private key blocks
- full terminal byte streams
- local-only dogfood recordings classified as `raw_local`

Public JSON dogfood, replay or demo artifacts are scanned for raw transcript
fields. Documentation can describe the policy, but generated evidence must carry
only summaries, scrubbed content or deterministic synthetic data.

When redaction rules change, every public artifact's `redaction_checked_at` must
be refreshed. Stale artifacts are validation failures, not silent warnings.

## Validation Commands

```text
cargo test -p terminal-fixtures m4_evidence
cargo test -p terminal-fixtures m4_compatibility
cargo test -p terminal-fixtures m4_performance
cargo test -p terminal-fixtures m4_replay
cargo test -p terminal-cli m4_performance_cli
cargo test -p terminal-cli m4_replay
cargo test -p terminal-cli m4_event_stream
cargo run -p terminal-cli -- validate-m4-evidence evidence/m4/evidence-manifest.json
cargo run -p terminal-cli -- validate-m4-compatibility evidence/m4/compatibility-matrix.json
cargo run -p terminal-cli -- verify-m4-replay crates/terminal-fixtures/fixtures/m4-replay --json-output evidence/m4/m4-replay-verification.json
cargo run -p terminal-cli -- export-m4-event-stream crates/terminal-fixtures/fixtures/m4-replay/basic-shell.json --output evidence/m4/replay-event-streams/basic-shell.jsonl
cargo run -p terminal-cli --release -- m4-benchmark --output evidence/m4/m4-benchmark-summary.json
cargo run -p terminal-cli --release -- m4-memory-profile --output evidence/m4/m4-memory-profile.json
cargo run -p terminal-cli --release -- m4-performance-report --bench evidence/m4/m4-benchmark-summary.json --memory evidence/m4/m4-memory-profile.json --thresholds evidence/m4/m4-performance-thresholds.json --json-output evidence/m4/m4-performance-report.json --markdown-output docs/m4-benchmarks-and-memory.md
```

The CLI loads the manifest, checks the schema, infers the repo root from
`evidence/m4/`, scans public artifacts and fails with the offending path and
manifest field.

The compatibility validator also checks linked fixtures. Any row claiming
`implemented` must have fixture-backed or replay-backed coverage, and Windows,
Linux and macOS measurement fields must be present in the machine-readable data.
